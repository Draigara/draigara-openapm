# OpenAPM Open Questions

**Status:** Living document

Only unresolved decisions belong here. A coding agent must not invent answers.

1. Which additional community packages should follow the five-package preview, and which behavioural evidence is required for each?
2. Should a future stable channel use the same marketplace identity and release tags, or a distinct stable catalogue?
3. How does the supported APM release represent package revocation and deprecation, and what maintainer procedure should follow?
4. When will APM `marketplace check` support direct immutable commit refs instead of requiring them to appear as advertised remote refs?

The Forge plugin preview tag is published. Preview publication now depends on clean production pack output and end-to-end acceptance evidence.
