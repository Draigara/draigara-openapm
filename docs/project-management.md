## Package proposal issue

Must contain:

- intended outcome and audience;
- why a package is needed;
- proposed upstream dependencies/assets;
- overlap with existing packages;
- security/runtime behaviour;
- licence/provenance;
- maintainer;
- acceptance scenarios;
- proposed metadata.

## Ownership

Every package has at least two code owners for stable publication where feasible. An unowned package is deprecated rather than left indefinitely unmaintained.
